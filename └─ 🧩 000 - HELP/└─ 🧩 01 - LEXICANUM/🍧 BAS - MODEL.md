# 🌸 MODEL

## 🧩 DEFINITION

Un `Model` est l’objet qui contient les données de l’application.  
Il peut être local (dans l’application) ou connecté à des services SAP via oData.

> [!TIP]
> Imagine le Model comme le réfrigérateur où sont stockés tous tes ingrédients (données), prêts à être utilisés.

## 🧩 UTILITE

Les Models fournissent les données aux Views via le Data Binding.  
Ils permettent de centraliser et de gérer toutes les informations de l’application.

> [!TIP]
> Concrètement, le Model stocke les données que tu affiches ou modifies dans tes écrans Fiori.

## 🧩 POINTS IMPORTANTS

- Peut être JSON, OData ou local.
- Connecté à la View via Data Binding.
- Facilite la récupération, l’affichage et la mise à jour des données.
- Partie centrale de l’architecture MVC dans UI5.

> [!TIP]
> Model = la source des données de ton application.
