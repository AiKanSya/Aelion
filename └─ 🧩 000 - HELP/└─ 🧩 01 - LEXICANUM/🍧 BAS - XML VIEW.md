# 🌸 VIEW (XML)

## 🧩 DEFINITION

Une `View` est la structure visuelle d’un écran dans une application UI5, souvent définie en XML.  
Elle contient tous les éléments graphiques : boutons, champs de saisie, tableaux, textes.

> [!TIP]
> Imagine la View comme le plan de ta maison, où tu décides où chaque meuble (bouton, tableau) sera placé.

## 🧩 UTILITE

La View organise ce que l’utilisateur voit et comment les éléments sont disposés.  
Elle sépare la présentation (View) du comportement (Controller).

> [!TIP]
> Concrètement, c’est le schéma que tu construis pour afficher correctement toutes les informations à l’écran.

## 🧩 POINTS IMPORTANTS

- Définie en XML.
- Associée à un Controller pour gérer le comportement.
- Peut être utilisée avec data binding pour afficher des données automatiquement.
- Partie essentielle de Fiori/UI5.

> [!TIP]
> La View = ce que l’utilisateur voit et comment c’est organisé sur l’écran.
