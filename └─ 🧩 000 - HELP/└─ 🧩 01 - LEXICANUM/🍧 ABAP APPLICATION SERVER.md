# 🌸 ABAP APPLICATION SERVER (AS)

## 🧩 DEFINITION

Un `ABAP Application Server (AS)` est un serveur qui exécute les programmes ABAP et gère les requêtes des utilisateurs.  
Il fait partie du Backend SAP et traite la logique métier, les calculs et les données.

> [!TIP]
> Imagine l’`AS` comme un chef qui suit des recettes (programmes ABAP) pour préparer les plats demandés par les clients (utilisateurs).

## 🧩 UTILITE

L’`AS` permet aux utilisateurs et aux applications Front-End (`Fiori/UI5`) de faire des opérations complexes dans SAP sans connaître les détails techniques.  
Il s’assure que les calculs et les traitements sont faits correctement et rapidement.

> [!TIP]
> Comme un robot cuisinier : tu donnes les ingrédients (données), il suit la recette et te renvoie le plat prêt (résultat du programme).

## 🧩 POINTS IMPORTANTS

- Contient la logique métier ABAP, les programmes, les fonctions et les services nécessaires au fonctionnement de SAP.
- Reçoit les requêtes du Front-End Server ou d’autres systèmes via RFC, oData ou autres protocoles.
- Peut être plusieurs dans un système SAP pour gérer la charge et assurer la disponibilité.

> [!TIP]
> l'`ABAP Application Server` = le "cœur" du traitement SAP : c’est là que toute la logique et les programmes sont exécutés.
