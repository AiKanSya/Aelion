# 🌸 UI5.YAML

> 🌺 Objectifs
>
> - [ ] Configurer le build et la structure de l’application UI5.
> - [ ] Définir les routes et les sources pour le tooling UI5.
> - [ ] Permettre la génération du build final et la prévisualisation.

## 🧩 DEFINITION

`ui5.yaml` est le fichier de configuration principal du UI5 Tooling.  
Il contient :

- Les sources de l’application (`webapp/`).
- Les dépendances externes UI5 (`resources`, `libs`).
- La configuration pour le build et la prévisualisation.

> [!TIP]  
> Pense au `ui5.yaml` comme à la carte de ton projet pour le UI5 Tooling : il indique où sont les fichiers et comment les traiter.

## 🧩 UTILITE

- Indiquer au UI5 Tooling où se trouvent les fichiers XML, JS, CSS et i18n.
- Configurer les middleware pour le mockserver, la preview ou les tests.
- Définir la destination du build (`dist/`) et la structure finale pour le déploiement.

> [!TIP]  
> Sans `ui5.yaml`, le tooling ne sait pas comment assembler ou prévisualiser l’application.

## 🧩 POINTS IMPORTANTS

- Contient la configuration du UI5 Tooling pour le build local ou pour le déploiement.
- Permet de définir des middleware pour simuler des services ou gérer les routes.
- Peut être différent pour chaque projet selon les besoins (mockserver, proxies, etc.).

## 🧩 EXEMPLE

```yaml
	specVersion: "2.2"
	rootProject: true
	type: application
	name: appdemofgi
	metadata:
	  version: 1.0.0
	resources:
	  configuration:
	    paths:
	      webapp: webapp
	builder:
	  customTasks:
	    - name: fiori-preload
	      afterTask: generateCachebusterInfo
	      configuration:
	        bundles:
	          - bundleDefinition:
	              bundleName: "Component-preload.js"
	              sections:
	                - mode: preload
	                  filters:
	                    - "/*.js"
	                    - "/*.xml"
	                    - "/*.json"
	                    - "/*.properties"
```
